## LLM Prompt: Rebuild BarterVerse Application From Scratch (Phased Approach)

**Objective:** Recreate the "BarterVerse" application, a Next.js-based bartering platform with AI-powered features. Follow the phases outlined below.

**Core Technologies:**
*   Next.js (App Router)
*   React (Functional Components, Hooks)
*   TypeScript
*   Tailwind CSS
*   ShadCN UI Components (for UI elements like Button, Card, Input, Dialog, etc.)
*   Genkit (for all AI-related backend logic, using Gemini models via `@genkit-ai/googleai`)
*   Lucide-React (for icons)
*   Date-fns (for date formatting)
*   Zod (for schema validation, especially in AI flows)

**General Guidelines:**
*   Use Server Components by default where appropriate.
*   Implement robust error handling and user-friendly loading states.
*   Prioritize clean, readable, and well-organized code.
*   All AI flows must be defined in `src/ai/flows/` and use the global `ai` object from `src/ai/genkit.ts`.
*   Data persistence will initially be mocked using `src/lib/dummy-data.ts` for users and items. AI configurations and logs will be stored in local `.json` or `.jsonl` files in the project root.
*   Adhere to Next.js best practices, including image optimization with `next/image` and `https://placehold.co` for placeholders with `data-ai-hint`.
*   Follow the UI styling guidelines previously established (ShadCN theme in `globals.css`, Tailwind semantic classes).

---
**Phase 1: Project Setup & UI Foundation**
1.  Initialize a new Next.js project configured with TypeScript, Tailwind CSS.
2.  Install and configure ShadCN UI. Set up `components.json`.
3.  Create a basic layout structure: `src/app/layout.tsx` (with `ClientLayoutWrapper` for Toaster and `GlobalFilterProvider`), `src/components/layout/Navbar.tsx`, `src/components/layout/Footer.tsx`.
4.  Define the application's color scheme and typography in `src/app/globals.css` (using HSL CSS variables for ShadCN theming) and `tailwind.config.ts` (font families: Belleza for headlines, Alegreya for body).
5.  Create basic versions of common UI components if not fully covered by ShadCN (or ensure ShadCN components are correctly aliased and available).

**Phase 2: Static Pages & Basic Navigation**
1.  Implement the homepage (`src/app/page.tsx`) with a basic structure for displaying item lists (placeholders for now).
2.  Create Sign In (`/auth/signin`) and Sign Up (`/auth/signup`) pages with forms (UI only, no authentication logic).
3.  Populate the Navbar with initial links: Home, List Item, Chats, Profile.
4.  Implement a basic Footer.

**Phase 3: Core Data Structures & Dummy Data**
1.  Define all necessary TypeScript types in `src/types/index.ts` for `Item`, `User` (including `UserProfilePreferences`, `UserStoredLocation`, `UserLogisticsPreferences`), `TradeOffer`, `ChatMessage`, `ItemLogistics`, `ItemTiming`, etc. Ensure all fields present in the final app are included.
2.  Create `src/lib/dummy-data.ts` and populate it with a few `dummyUsers` and `dummyItems`. Include functions like `addNewItemToDummyData` and `updateUserPreferencesInDummyData`.

**Phase 4: Core Bartering Feature UIs (Display & Forms)**

2.  **Item Detail Page (`/items/[id]`):**

3.  **User Profile Page (`/profile/[userId]` and `/profile/me`):**
    *   Display user information: avatar, name, rating, trades, bio.
    *   List items offered and items wanted by the user.
    *   Display user's trading preferences and logistics preferences.
4.  **Homepage (`/app/page.tsx`):**
    *   Implement UI to display a list of all available items from other users.
    *   Include structure for "AI Suggested Matches" for the current user's items (will be populated in a later phase).
5.  **Chats Page (`/chats`):**

    *   Section for "Direct Assistant Chat".
6.  **Trades Overview Page (`/trades`):**

**Phase 7: AI - Item Matching & Rationale**
1.  Implement `itemMatchFlow.ts`. Start with a "simple" matching logic (keyword, category based). Then, implement the "advanced" logic considering `listingType` (offer vs. want), `isGiftItForward`, `openToAnyOpportunity`, user preferences (stubbed for now if not fully available), and reciprocity (identifying if User B has an item User A wants - `reciprocalItemId`).

3.  Create and integrate the `SuggestedMatches.tsx` component on the Item Detail page (`/items/[id]`) to show matches for the viewed item (if not owned by current user).
